# Многопоточность: жизненный цикл

Жизненный цикл многопоточности в Java описывает стадии жизни, которые проходит поток при создании, выполнении и завершении своей работы. В Java создание и управление потоками осуществляется с помощью класса Thread.

1) Создание потока

Для создания потока в Java необходимо создать экземпляр класса Thread и передать в конструктор объект, реализующий интерфейс Runnable или Callable. Runnable является простым функциональным интерфейсом, определяющим метод run(), который выполняет основную работу потока. Callable является также функциональным интерфейсом, который возвращает результат выполнения работы потока.

2) Запуск потока

После создания экземпляра потока необходимо вызвать метод start(), который запускает новый поток и вызывает метод run(). Метод run() содержит основной код выполнения работы потока.

3) Выполнение потока

Поток начинает свою работу с метода run(), который содержит основную работу потока. В зависимости от того, сколько потоков выполняется одновременно и от приоритета потока, операционная система распределяет вычислительные ресурсы между потоками.

4) Завершение потока

Поток завершается после выполнения метода run() или при возникновении исключения. После завершения потока его нельзя запустить повторно, поэтому для повторного выполнения задачи необходимо создать новый экземпляр класса Thread.

5) Управление потоками

Java предоставляет набор методов для управления потоками, таких как sleep(), yield(), join(), interrupt(), isAlive(), setName() и другие.

Метод sleep() приостанавливает выполнение потока на определенное время.

Метод yield() передает управление другому потоку с тем же или большим приоритетом.

Метод join() ожидает завершения работы потока, на который ссылается вызывающий поток.

Метод interrupt() прерывает выполнение потока.

Метод isAlive() возвращает true, если поток все еще выполняется.

Метод setName() устанавливает имя потока.
