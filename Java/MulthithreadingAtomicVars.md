# Атомарные переменные

Атомарные переменные в Java - это переменные, которые обеспечивают атомарность операций над ними. Атомарность означает, что операции чтения и записи выполняются за один шаг, так что другой поток не может получить доступ к переменной в промежутке между этими операциями.

В Java атомарные переменные реализованы через классы java.util.concurrent.atomic. Некоторые из таких переменных: AtomicInteger, AtomicLong, AtomicBoolean, AtomicReference и другие.

Атомарные переменные используются, когда необходимо обеспечить безопасность работы с общими ресурсами в многопоточных приложениях. Это может быть полезно, например, для реализации счетчиков, которые используются несколькими потоками одновременно.

Если переменная не является атомарной, то при параллельном доступе к ней из разных потоков могут возникнуть ошибки, например, состояние переменной может оказаться некорректным или изменения одного потока могут быть потеряны при работе другого потока.

Таким образом, атомарные переменные позволяют гарантировать безопасность работы с общими ресурсами в многопоточных приложениях.

Пример:
```
public class Main {

    static AtomicInteger atomicInteger = new AtomicInteger(0);

    public static void main(String[] args) throws InterruptedException {
        for (int i = 0; i < 10000; i++) {
            new MyThread().start();
        }
        Thread.sleep(1000);
        System.out.println(atomicInteger.get());
    }

    static class MyThread extends Thread {
        @Override
        public void run() {
            atomicInteger.incrementAndGet();
        }
    }
}
```

За счет использования атомарного инта, мы можем быть увереными что в даном примере вывод всегда будет 10 000.
