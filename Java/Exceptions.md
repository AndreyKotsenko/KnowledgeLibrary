# Исключения 

Exceptions (исключения) - это механизм в языке программирования Java, который позволяет обрабатывать ошибки во время выполнения программы.

Если при выполнении программы возникает ошибка (например, деление на ноль или попытка доступа к несуществующему объекту), то создается объект типа Exception, который может быть обработан в коде программы.

Java предоставляет множество встроенных классов исключений, которые могут быть использованы для обработки различных типов ошибок. Кроме того, вы можете создавать свои собственные исключения, которые будут использоваться в вашей программе.

Для обработки исключений в Java используются блоки try-catch. Код, который может вызвать исключение, помещается в блок try, а код, который будет обрабатывать исключение, помещается в блок catch. Если исключение не было обработано в блоке catch, то оно передается дальше по стеку вызовов до тех пор, пока не будет обработано или программа не завершится.

В Java исключения делятся на две категории: проверяемые (checked) и непроверяемые (unchecked).

Проверяемые исключения - это исключения, которые должны быть обработаны в коде программы, либо переданы выше по стеку вызовов. Если в методе возникает проверяемое исключение, то метод должен либо обработать его при помощи конструкции try-catch, либо объявить, что метод может выбросить исключение с помощью ключевого слова throws. Примеры проверяемых исключений: FileNotFoundException, IOException, SQLException.

Непроверяемые исключения - это исключения, которые не обязательно должны быть обработаны в коде программы. Это обычно ошибки программиста или проблемы с окружением исполнения программы. Примеры непроверяемых исключений: NullPointerException, ArrayIndexOutOfBoundsException, ClassCastException.

При использовании непроверяемых исключений, компилятор не требует обработки исключений, но это не значит, что их не нужно обрабатывать. Непроверяемые исключения, как и проверяемые, могут привести к остановке программы, если их не обрабатывать.

Обычно, проверяемые исключения возникают в связи с вводом-выводом данных, работой с файлами и сетевыми соединениями, а непроверяемые исключения связаны с ошибками в работе с памятью, типами данных и др.
