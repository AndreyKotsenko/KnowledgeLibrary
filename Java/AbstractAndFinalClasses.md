# Абстрактные классы

Абстрактный класс в Java - это класс, который содержит один или несколько абстрактных методов (методов без реализации) и может содержать методы с реализацией. Абстрактные классы используются для определения общего поведения для группы классов.

Абстрактные методы объявляются с помощью ключевого слова abstract и не имеют тела метода. Они должны быть реализованы в дочерних классах, наследующих абстрактный класс. Если дочерний класс не реализует абстрактный метод, то он также должен быть объявлен как абстрактный.

Абстрактный класс не может быть создан напрямую с помощью оператора new, поскольку он не имеет полной реализации. Однако, он может быть использован в качестве типа переменной, что позволяет ссылаться на любой объект-наследник абстрактного класса.

Абстрактные классы могут также содержать конструкторы, обычные методы, переменные экземпляра и статические переменные.

Использование абстрактных классов позволяет избежать дублирования кода в дочерних классах, обеспечивает возможность использовать полиморфизм и упрощает поддержку кода в случае изменений в иерархии наследования.

Пример:
```
abstract public class Car {
    int speed;
    abstract void Run();
    void Stop() {
        speed = 0;
    }
}

class Ferrari extends Car {

    @Override
    void Run() {
        speed = 25;
    }
}
```
Здесь мы создали 2 класса, один абстрактный и один дочерний.

Функция main выглядит так:
```
    public static void main(String[] args) {
        //Car car = new Car(); we can't do this because this class is abstract

        Car test = new Ferrari(); // we can do this because Ferrari is not abstract

        test.Run();
        System.out.println(test.speed);
        test.Stop();
        System.out.println(test.speed);
    }
```

**final** в Java - это ключевое слово, которое используется для указания, что переменная, метод или класс не может быть изменен после инициализации. Значение переменной final должно быть установлено при ее объявлении, и оно не может быть изменено позднее.

В случае переменных, объявленных как final, они должны быть инициализированы перед использованием, и попытка присвоения нового значения будет приводить к ошибке компиляции.

Для методов, объявленных как final, нельзя изменять их реализацию в дочерних классах. Это гарантирует, что метод будет работать одинаково во всех классах, которые наследуют этот метод.

Для классов, объявленных как final, нельзя наследовать от них, то есть они являются конечными и не могут иметь подклассов.

Использование ключевого слова final может улучшить производительность программы, облегчить ее понимание и сделать ее более безопасной, так как исключается возможность изменения значения или поведения.

Пример:
```
final class Test{

    int a = 10;
}

//class SampleText extends Test {} // error because the father class is final

class Simple {
    final int a = 10;

    final void getSmt(){
        System.out.println("I am final");
    }

    void printHi() {
        System.out.println("Hi");
    }
}

class FirstSimple extends Simple {

    @Override
    void printHi() {
        System.out.println("Hello");
    }

//    @Override
//    void getSmt(){                          // error because method is final in father class
//        System.out.println("I am final");
//    }

}
```
