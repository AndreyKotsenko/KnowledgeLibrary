# Git
Git - это система контроля версий. Ее назначение это ввести базу данных со всеми изминениями репозитория. В этой базе данных по факту хранится вся история проекта. Так же гит позволяет команде работать над одним проектом.

# Системы контроля версий
### Локальная система контроля версий
Многие из нас вели свою локальную систему контроля версий. По факту это просто сохранение проекта на определенной стадии на своем жестком диске в папках. После каждого изминения мы создаем отдельную директорию в которую помещаем новую версию проекта. 

Видимые минусы: 
 - Неудобность. 
 - Все хранится в одном месте (жесткий диск)

### Централизованные системы контроля версий
В этом случае у нас есть один сервер на котором хранится репозиторий и некоторые люди имеют к нему доступ. За счет этого теперь команда может вместе работать над одним проектом.

Огромная проблема:
 - Все хранится в одном месте 

Из-за этой проблемы, если у нас повредится сервер, то вся история будет удалена. Остануться только проекты на локальных компах разрабов с последней ревизией(версией).
### Распределенные системы контроля версий
И теперь перейдем к распределенной системе, которой и является Git.  В РСКВ клиенты не просто скачивают снимок всех файлов (состояние файлов на определённый момент времени) — они полностью копируют репозиторий. В этом случае, если один из серверов, через который разработчики обменивались данными, умрёт, любой клиентский репозиторий может быть скопирован на другой сервер для продолжения работы. Каждая копия репозитория является полным бэкапом всех данных. Более того, многие РСКВ могут одновременно взаимодействовать с несколькими удалёнными репозиториями, благодаря этому вы можете работать с различными группами людей, применяя различные подходы единовременно в рамках одного проекта. Это позволяет применять сразу несколько подходов в разработке, например, иерархические модели, что совершенно невозможно в централизованных системах.
